# BUSINESS_LOGIC.md — Бизнес-логика NeiroUstaz

## 1. Аутентификация

### Регистрация
- Поля: ФИО, email или телефон, пароль
- Пароль хэшируется через `bcryptjs` (saltRounds = 10)
- Возвращает JWT токен
- Дефолтный язык: `ru`

### Логин
- Сравнение пароля через `bcrypt.compare`
- Возвращает JWT со сроком `JWT_EXPIRES_IN` из .env
- Токен отправляется в заголовке `Authorization: Bearer <token>`

### Защита маршрутов
- `authMiddleware` декодирует токен и добавляет `req.user` к запросу
- Если токен истёк или неверен — 401 Unauthorized

## 2. Профиль пользователя

Доступные поля для редактирования:
- ФИО
- Университет
- Курс/год обучения
- Язык интерфейса (kz/ru)

Автоматически рассчитываемые поля:
- `points` — сумма всех начисленных баллов
- `level` — рассчитывается на основе баллов

### Система уровней

| Уровень | Название (RU) | Название (KZ) | Баллы |
|---------|--------------|--------------|-------|
| 1 | Студент | Студент | 0–99 |
| 2 | Педагог | Мұғалім | 100–299 |
| 3 | Эксперт | Сарапшы | 300+ |

## 3. Система баллов (Points)

### Начисление баллов

| Действие | Баллы |
|----------|-------|
| Правильный ответ в тесте | +10 |
| Завершение мини-кейса | +15 |
| Первый пост в сообществе | +5 |
| Ответ на чужой вопрос (5+ лайков) | +10 |
| Ежедневное задание (стрик) | +5 бонус |

### Правила
- Баллы не сгорают
- За неправильный ответ баллы не снимаются
- Одно задание в день — повторное выполнение не засчитывается
- Баллы отображаются в профиле пользователя

## 4. Ежедневные задания (Tasks)

### Генерация
- ИИ ежедневно генерирует одно задание для каждого пользователя (или одно общее)
- Два типа: `test` (тест с вариантами) и `case` (мини-кейс)
- Задание хранится в БД на текущую дату
- Генерируется на обоих языках (kz + ru)

### Структура задания (JSON)

```json
// Тип: test
{
  "type": "test",
  "question": "Ребёнок с аутизмом не реагирует на имя. Какое первое действие педагога?",
  "options": [
    "Повторить имя громче",
    "Использовать визуальный контакт и жест",
    "Сообщить родителям",
    "Продолжить урок"
  ],
  "correct": 1,
  "explanation": "При РАС визуальный контакт и физический жест эффективнее звукового обращения..."
}

// Тип: case
{
  "type": "case",
  "situation": "На уроке ребёнок с ЗПР начал плакать и отказывается работать...",
  "question": "Опишите ваши действия как педагога",
  "hints": ["Подумайте о причине срыва", "Вспомните правила коррекционной педагогики"],
  "ideal_answer": "...",
  "criteria": ["Эмпатия", "Коррекционный подход", "Коммуникация с классом"]
}
```

### Проверка ответов
- Тест (`test`): сравнивается `answer.selected` с `correct` — просто
- Кейс (`case`): ответ пользователя отправляется в Groq API (LLaMA 3), который оценивает по критериям и даёт обратную связь

## 5. ИИ-помощник (Chat)

### Роль ИИ (системный промпт)
```
Ты — педагогический ИИ-помощник для студентов, изучающих инклюзивное образование.
Ты помогаешь:
- Отвечать на вопросы об инклюзивном образовании, аутизме, ЗПР, ОНР и других ООП
- Составлять характеристики на детей
- Создавать планы уроков
- Анализировать педагогические ситуации

Принципы:
- Не давай готовых ответов на задания платформы — задавай наводящие вопросы
- Развивай критическое педагогическое мышление
- Используй язык пользователя (казахский или русский)
```

### Контекст чата
- Сохраняются последние **20 сообщений** из `ai_chat_history`
- При каждом запросе история загружается и передаётся в Groq API
- Старые сообщения (старше 30 дней) могут быть автоматически удалены

### Что умеет ИИ-помощник
- Отвечать на вопросы по инклюзивному образованию
- Генерировать характеристику на ребёнка (по введённым данным)
- Составлять план урока / адаптированный материал
- Помогать придумывать игры для детей с ООП
- Объяснять термины (РАС, ЗПР, ОНР, АООП и т.д.)

## 6. Сообщество

### Посты
- Пользователь публикует вопрос или мысль (текст)
- Пост привязан к языку (kz/ru) — фильтр по языку на фронте
- Сортировка: по дате (новые) или по лайкам (популярные)

### Лайки
- Один пользователь — один лайк на один пост
- Таблица `post_likes` предотвращает дублирование
- Toggle-лайк: если уже лайкнул — убирает лайк

### Ответы на посты
- Ответ — это отдельный пост с `parent_id` (опционально, для упрощения)
- Или просто комментарии как дочерние записи

## 7. Геймификация / Достижения

### Достижения (примеры)

| Код | Название | Условие |
|-----|---------|---------|
| `first_task` | Первый шаг | Выполнить первое задание |
| `streak_3` | 3 дня подряд | Задания 3 дня подряд |
| `streak_7` | Неделя педагога | Задания 7 дней подряд |
| `community_first` | Голос в сообществе | Первый пост |
| `expert_level` | Эксперт | Достичь 3-го уровня |

### Проверка достижений
- Вызывается после каждого действия (задание, пост, лайк)
- Сервис `checkAchievements(userId)` проверяет условия
- Если условие выполнено и достижение ещё не выдано — записывает в `user_achievements`

## 8. Двуязычность

### Принцип работы
- Язык хранится в профиле пользователя (`user.lang = 'ru' | 'kz'`)
- `langMiddleware` добавляет `req.lang` к каждому запросу
- Сервисы возвращают контент на нужном языке: `content[req.lang]`
- Задания и обратная связь хранятся сразу в двух полях: `content_ru`, `content_kz`

### Промпты для ИИ
- Системный промпт включает инструкцию отвечать на языке пользователя
- Язык явно передаётся в промпт: `"Отвечай на казахском языке"`
